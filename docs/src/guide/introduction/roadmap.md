# 路线图

我们会持续把 NG Gateway 打磨成一个**稳定、可靠、可运维、易扩展**的工业级 IoT 网关产品。

---

## Milestone M0：稳定可上线

- **可靠运行与性能基线**
  - 稳定性：弱网/抖动/高负载场景下的稳定运行与自动恢复
  - 性能：形成官方压测基线（点位规模 × 周期 × 延迟），并持续优化吞吐/延迟
- **可观测与排障（更好用）**
  - 指标与仪表盘完善（南向通道 / 北向 App / 背压队列 / 资源）
  - 排障工具箱：连接测试、写点/Action 调试、临时日志级别（自动回滚）等
- **配置与变更治理**
  - 配置变更审计（谁改了什么、什么时候、影响范围）
  - 配置快照与回滚（降低现场变更风险）

---

## Milestone M1：弱网可靠交付 + 可控降级

- **断网续传（WAL + 回放）**
  - 弱网/断链/掉电场景下尽量保障关键数据完整交付
  - 支持磁盘缓冲、配额与清理策略、回放限速与实时链路隔离
- **队列满智能降级（统一策略）**
  - 当下游拥塞时以可控方式降级，优先保护网关稳定性与关键业务
  - 支持 TTL、采样、按 last 合并、优先级保留（告警/控制优先）等策略

---

## Milestone M2：南北向生态扩展 + 产品化治理

### 南向生态（Southward Drivers）

- **增强现有驱动（已支持，但会持续优化）**
  - Modbus RTU/TCP：批量采集规划、性能优化、诊断能力增强
  - Siemens S7：更多型号/更多数据类型、容错与现场排障增强
  - IEC 60870-5-104：更强容错、事件/遥控场景增强
  - DL/T 645（1997/2007）：更多表计与异常场景覆盖
  - OPC UA Client：订阅/采集策略、稳定性与诊断增强
  - Serial（RS-232/485）、TCP/UDP：连接治理、超时与重试策略优化
- **新增南向驱动（计划扩展，按需求优先级推进）**
  - 工业自动化/PLC：EtherNet/IP（CIP）、Mitsubishi MC、Omron FINS、Profinet（评估）
  - 电力/能源：IEC 61850（MMS/报告；GOOSE 评估）、DNP3
  - 楼宇/表计/总线：BACnet、M-Bus / wM-Bus、CAN / CANopen
- **通用能力持续增强**：统一超时/重试/退避、批量规划、并发能力、现场诊断工具链

### 北向生态（Northward Plugins）

- **增强现有插件（已支持，但会持续优化）**
  - ThingsBoard：连接稳定性、上/下行链路能力、诊断体验持续增强
  - Kafka / Pulsar：批量/压缩、背压与重试策略、吞吐与延迟优化
  - OPC UA Server：数据模型映射、写入链路与可观测性增强
- **新增北向插件（计划扩展，按需求优先级推进）**
  - IoT 平台/云厂商：AWS IoT Core、Azure IoT Hub、阿里云 IoT / 腾讯云 IoT / 华为云 IoTDA（按需）
  - 消息与流式：MQTT v5、NATS / JetStream、RabbitMQ（AMQP）、Redis Streams
  - 数据落库/分析型目的端：InfluxDB、TimescaleDB / PostgreSQL、ClickHouse、Elasticsearch / OpenSearch、S3 / MinIO（按场景）
- **更高吞吐的编码与传输**
  - 二进制 payload（Protobuf/Avro 等）以降低带宽与 CPU（适用于高频/大规模点位）
  - 更丰富的映射与协议适配能力（在不牺牲性能前提下提升灵活性）

### 产品化治理

- **插件/驱动治理**
  - 兼容性与升级策略（版本约束明确、升级可预测）
  - 灰度与回滚（出问题可快速退回）
  - 签名与来源校验（供应链安全）
- **安全增强**
  - 管理面安全基线持续强化（权限、审计、敏感信息保护）
  - TLS/mTLS 能力增强（按场景逐步覆盖）

---

## 我们长期坚持的迭代方向

- **性能优先**：高吞吐、低延迟、低资源占用
- **稳定性优先**：弱网容错、可恢复、可解释
- **可运维优先**：指标/日志/诊断闭环，降低现场成本
- **生态优先**：南向驱动 + 北向插件的长期扩展与工程化交付
